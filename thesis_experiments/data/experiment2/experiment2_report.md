# 实验2：多模型排序融合对比报告

生成时间: 2026-05-19 17:07:12

## 1. 实验设计

| 实验组 | 融合策略 | 核心思路 |
|--------|---------|---------|
| B-LGBM | 无融合（单模型） | LightGBM基线 |
| B-XGB | 无融合（单模型） | XGBoost基线 |
| E2a | 分数平均融合 | 多模型预测分数归一化后取平均 |
| E2b | RRF融合（k=60） | Reciprocal Rank Fusion，按排名倒数加权 |
| E2c | Stacking融合 | Ridge元学习器在验证集上学习最优融合权重 |
| E2d | 加权RRF融合 | 根据验证集NDCG@10为各模型分配RRF权重 |

## 2. 融合参数

- RRF平滑常数 k=60
- Stacking元学习器系数: LightGBM=-0.001163, XGBoost=-0.007508
- Stacking截距: -0.00031590443540992306
- NDCG@10权重: LightGBM=0.1629, XGBoost=0.1580

## 5. 关键论证逻辑

### 5.1 实验假设

单模型排序存在模型特定偏差，多模型融合可以通过互补性提升排序质量：
- **E2a（分数平均）**：简单等权平均，假设各模型预测能力相当
- **E2b（RRF）**：按排名倒数加权，对分数尺度不敏感，预期换手率更低
- **E2c（Stacking）**：在验证集上学习最优线性组合权重
- **E2d（加权RRF）**：根据NDCG@10为各模型分配不同权重

### 5.2 预期效果

1. RRF融合因基于排名而非分数，应降低组合换手率
2. Stacking融合因学习了验证集上的最优权重，应提升排序质量
3. 加权RRF应优于等权RRF（NDCG更高的模型获得更大权重）
4. 分数平均融合作为最简单的融合方式，作为融合基线

### 5.3 实验结论

E2b-RRF融合**未优于** B-LGBM基线，可能原因：
- 两模型排序高度相关，融合增益有限
- RRF的排名聚合可能平滑掉了单模型的排序优势
- 验证集与测试集分布差异导致Stacking权重过拟合
