# 项目名称：基于排序学习的量化投资选股策略设计与应用 整体开发阶段规划（最终版）

## 项目技术栈总览
- **前端技术栈**：React 18 + TypeScript + Vite + ECharts/Plotly（数据可视化） + TailwindCSS（UI样式）  
- **后端技术栈**：Python 3.8+ + FastAPI + Pandas/Numpy + LightGBM + XGBoost + akshare（数据源）+ DeepSeek API（AI推荐服务）
- **数据库**：SQLite（存储预处理后的因子数据、回测结果、用户影子账户数据、AI建议记录）
- **核心依赖**：lightgbm, xgboost, pandas, numpy, akshare, fastapi, uvicorn, plotly, echarts, react-router-dom, axios, tailwindcss, openai（兼容DeepSeek聚合站接口）
- **部署**：Zeabur（统一部署前后端）

- **预估任务数量**：9个

### 阶段2：数据层开发（数据处理、因子计算、特征工程）
- **里程碑**：完成固定股票池的周频截面数据生成，计算全部基础因子，生成排序学习所需的训练/验证/测试数据集（query格式），并实现实时数据更新模块
- **核心目标**：
  1. 实现股票池筛选（近10年存续A股约3000只），获取并清洗日线数据（前复权、停牌标记、涨跌停标记）。
  2. 将日线数据对齐为周频截面（每周最后一个交易日），生成基础价格、成交量特征。
  3. 计算10~30个基础因子（动量、波动、流动性、技术指标等），保证因子无未来信息。
  4. 构造排序学习数据集：按时间切分训练/验证/测试集，每个截面为一个query group，标签为未来一周收益率，特征为因子值。
  5. 实现实时数据获取模块：获取最新交易日数据，计算因子，生成用于实盘建议的当前截面数据。
  6. 将处理好的数据存储为SQLite/Parquet，并编写数据加载工具函数。
- **核心交付物**：
  - 历史日线数据存储（SQLite/Parquet）
  - 周频截面数据表（包含股票代码、日期、所有因子、未来一周收益率）
  - 按query group划分的训练/验证/测试集文件
  - 实时数据更新模块（`src/realtime_updater.py`）
  - 数据加载模块（`src/data_loader.py`）
- **阶段核心执行模型**：
  - GLM-5：数据处理主流程、因子计算逻辑、时间切分策略、实时更新逻辑
  - DeepSeek-V3.1-Terminus：因子计算代码性能优化
- **预估任务数量**：7个

### 阶段3：模型层开发（LightGBM + XGBoost 排序学习）
- **里程碑**：完成LightGBM和XGBoost排序学习模型的训练、调优、保存，输出模型评估指标，并实现在线推理接口
- **核心目标**：
  1. 实现LightGBM的LambdaRank训练流程：按query group加载数据，设置`lambdarank`目标，调优超参数。
  2. 实现XGBoost的`rank:ndcg`训练流程，保持相同的数据组织方式。
  3. 在验证集上评估两个模型的排序性能（NDCG@K、MAP），并进行超参数搜索。
  4. 输出特征重要性，保存模型文件（`.pkl`或`.txt`）。
  5. 编写模型预测模块（`src/predictor.py`），支持对任意新截面数据输出股票得分。
  6. 为后续AI推荐服务预留模型结果输出接口（如返回TopN股票、得分、特征重要性等）。
- **核心交付物**：
  - 训练脚本（`src/model_lightgbm.py`、`src/model_xgboost.py`）
  - 训练好的模型文件（`models/lightgbm.pkl`、`models/xgboost.pkl`）
  - 预测器模块（`src/predictor.py`）
  - 模型评估报告（含特征重要性JSON、NDCG曲线数据）
- **阶段核心执行模型**：
  - GLM-5：模型训练核心逻辑、调优流程、预测接口
  - Kimi K2.5：跨文件依赖分析、模型保存与加载规范校验
- **预估任务数量**：5个

### 阶段4：回测与评估框架开发
- **里程碑**：实现符合行业标准的回测引擎（T+1、税费、滑点、观望操作），生成策略收益曲线、各项指标，支持两个模型的对比输出，并存储回测结果供前端使用
- **核心目标**：
  1. 构建回测引擎：使用已训练好的固定模型进行滚动预测（每周加载模型预测下周），严格实现T+1交易规则、印花税（卖出单边0.05%）、佣金（双边0.03%）、滑点（双边0.1%）。支持空仓（观望）逻辑。
  2. 实现选股逻辑：根据模型得分选取Top N股票等权配置，计算持仓收益，考虑涨停无法买入、跌停无法卖出等约束。
  3. 计算策略评估指标：年化收益、夏普比率、最大回撤、换手率、胜率、NDCG@K等。
  4. 生成回测结果数据：每日净值序列、持仓明细、指标值，存储为JSON格式供前端使用。
  5. 实现LightGBM与XGBoost策略对比功能，输出对比图表数据。
- **核心交付物**：
  - 回测引擎模块（`src/backtest.py`）
  - 回测结果存储（`data/backtest_results/`）
  - 指标计算工具函数（`src/metrics.py`）
  - 生成前端数据脚本（`src/generate_website_data.py`）输出JSON文件
- **阶段核心执行模型**：
  - GLM-5：回测逻辑、指标计算、滚动预测实现、交易成本与T+1约束
  - DeepSeek-V3.1-Terminus：回测性能优化
- **预估任务数量**：5个

### 阶段5：AI推荐服务与影子账户功能开发
- **里程碑**：实现DeepSeek AI推荐服务、影子账户管理、实时持仓建议，提供流式输出特定格式的Markdown建议给前端，后端计算买卖价格区间
- **核心目标**：
  1. 实现影子账户模块：用户可设置回测时间范围和预测时间范围，输入自定义持仓（股票代码、数量、成本价），系统存储账户状态（SQLite，无鉴权，最多10人使用）。
  2. 实现实时建议接口：获取最新交易日数据，调用预测器输出当前Top N股票得分，并结合用户影子账户持仓，构造提示词。
  3. 后端计算买卖价格区间：基于当前价、历史波动率（如ATR）、近期高低点等，计算出合理的买入参考价区间和卖出目标价区间，作为提示词的一部分。
  4. 接入DeepSeek API（聚合站自定义URL+密钥）：将模型预测结果（得分前10股票、特征重要性）、用户持仓数据、市场价格区间等信息封装为提示词，调用DeepSeek API获取投资建议，要求返回特定格式的Markdown（包含操作建议、风险提示等）。
  5. 实现流式响应（Server-Sent Events），前端实时渲染Markdown建议。
  6. 存储用户影子账户数据和AI建议记录（SQLite），支持历史查看。
- **核心交付物**：
  - 影子账户管理模块（`src/account.py`）
  - 价格区间计算工具（`src/price_range.py`）
  - 提示词构造器（`src/prompt_builder.py`）
  - DeepSeek流式接口（`src/deepseek_stream.py`）
  - 实时建议API端点（`/api/realtime_advice`）
  - 相关数据库表结构
- **阶段核心执行模型**：
  - GLM-5：后端业务逻辑、API设计、提示词工程、数据库操作、价格区间算法
  - Kimi K2.5：跨模块集成分析、API调用链校验
- **预估任务数量**：7个

  1. 基于React + TypeScript搭建前端工程，实现路由、状态管理（Zustand）、API请求。
  2. 根据阶段1制定的UI规范，开发全局布局、导航栏、图表组件。
  3. 实现“策略概览”页面：展示项目简介、模型架构图、数据范围。
  4. 实现“回测仪表盘”页面：使用ECharts/Plotly绘制收益曲线、月度热力图、持仓变动图，展示关键指标卡片，支持LightGBM/XGBoost切换对比。

## 需求确认与补充说明（已整合）
- **DeepSeek API**：使用聚合站提供的URL和密钥，封装为兼容OpenAI接口的客户端，支持流式输出。  
- **买卖价格区间**：后端负责计算（如基于ATR、支撑阻力位），提供数据给AI提示词，AI在此基础上生成更精准的建议。  
- **影子账户**：无登录鉴权，单用户或多用户通过本地存储或简单账户名区分（最多10人）。  
- **AI回复格式**：要求返回特定Markdown结构（如“操作建议”、“买入区间”、“卖出区间”、“风险提示”等），前端按格式渲染。  
- **Zeabur部署**：按照常规Python FastAPI + 静态文件部署方式配置。  
- **回测引擎**：严格遵循行业主流框架，包含T+1、印花税、佣金、滑点、涨停/跌停限制、观望（空仓）逻辑。
