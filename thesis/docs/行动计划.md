## 第1天：修复数据读取问题

**问题**：数据文件是 `train.parquet.gz`，但 `data_loader.py` 读取的是 `train.parquet`。

**做法**：把 `.parquet.gz` 解压为 `.parquet`，这是最快的修复方式。

```bash
cd d:\量化\V2.0\backend
python -c "
import gzip, shutil
for name in ['train', 'val', 'test']:
    src = f'data/{name}.parquet.gz'
    dst = f'data/{name}.parquet'
    with gzip.open(src, 'rb') as f_in:
        with open(dst, 'wb') as f_out:
            shutil.copyfileobj(f_in, f_out)
    print(f'{src} -> {dst} done')
"
```

**验证**：
```bash
python -c "import pandas as pd; print(pd.read_parquet('data/train.parquet').shape)"
```
应输出 `(766196, 63)`。

# 二、第2周：实现创新点①（收益-夏普感知损失函数）

## 第6-7天：实现收益加权标签

**最简单的实现方式**：修改 `future_return_to_relevance` 函数，在标签中融入收益率信息。

**为什么改标签而不是改损失函数**：LightGBM的自定义目标函数需要手写梯度和Hessian，实现复杂且容易出错。修改标签的效果类似，但实现简单得多。

**具体做法**：在 `model_lightgbm.py` 中新增 `return_aware_relevance` 函数（代码已在实验方法论.md中给出），然后在训练时用这个函数替代原来的 `future_return_to_relevance`。

**验证**：
```bash
python -m src.model_lightgbm --trials 20
```
确认模型能正常训练，验证集NDCG@10不低于原来的80%。

## 第8-9天：实现夏普惩罚

**做法**：在超参搜索的目标函数中，除了NDCG@10，还加入验证集Top-K组合的模拟夏普比率：

```python
def optuna_objective(trial):
    # ... 原有超参搜索代码 ...

    # 标准NDCG评估
    ndcg_score = evaluate_ndcg(model, val_data)

    # 新增：验证集Top-K模拟夏普
    sharpe_score = evaluate_topk_sharpe(model, val_data, top_k=20)

    # 组合目标：NDCG + λ * Sharpe
    combined = ndcg_score + 0.1 * max(0, sharpe_score)

    return combined
```

**验证**：训练完成后跑回测，对比：
- 标准NDCG损失：年化收益X%，夏普Y
- 收益加权标签：年化收益X+5~15%，夏普Y+0.1~0.3
- 收益加权+夏普惩罚：年化收益X+10~20%，夏普Y+0.2~0.5
