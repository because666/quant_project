# 一、核心问题回答

## 1.1 "自己搭建的回测框架会不会没有说服力？"

**短答案：不会，但需要补几个关键环节。**

**长答案：**

学术界并不要求使用特定回测工具。你的自研框架已经具备了学术论文认可的核心要素：

| 要素 | 你的框架 | 学术要求 | 是否达标 |
|------|---------|---------|---------|
| T+1约束 | 周频调仓天然满足 | 必须显式声明 | ✅ 达标 |
| 交易成本 | 佣金0.03%+印花税0.05%+滑点0.1% | 必须显式建模 | ✅ 达标 |
| 涨跌停约束 | 涨停不买、跌停不卖 | 必须实现 | ⚠️ 部分达标（见下文） |
| 空仓支持 | 支持 | 应支持 | ✅ 达标 |
| 时间切分 | train/val/test按时间切分 | 严格时序切分 | ✅ 达标 |
| 无前视偏差 | 因子shift(1)、模型固定 | 必须避免 | ✅ 基本达标 |

**但你的框架存在3个必须修复的问题**（详见第三部分），修复后即可达到学术标准。

**真正让回测有说服力的不是工具，而是方法论。** 审稿人看的是：

1. 你是否遵循了正确的实验流程（时间切分、避免泄露）
2. 你是否做了统计检验（不能只看数字大小，要看是否显著）
3. 你是否做了消融实验（证明每个创新点都有贡献）
4. 你是否做了稳健性检验（换参数、换时间段结果是否一致）

## 1.2 "实验怎么做？总不能实盘吧？"

**学术论文不需要实盘。** 量化选股论文的标准做法是**回测（Backtesting）**，而非实盘交易。

学术界认可的实验验证层次：

```
第一层：排序质量评估（NDCG@K、MAP）     ← 必做，证明模型排序能力强
第二层：回测绩效评估（年化收益、夏普、回撤） ← 必做，证明策略能赚钱
第三层：统计显著性检验（t检验、Bootstrap）  ← 必做，证明结果不是偶然
第四层：稳健性检验（换参数、换时段）       ← 强烈建议，证明结果不依赖特定设置
第五层：消融实验（逐步移除创新组件）       ← 必做，证明每个创新点都有贡献
第六层：实盘验证                          ← 不要求，但如果有会大大加分
```

你的论文做到前五层就足够了。

# 三、回测框架必须修复的3个问题

## 3.1 问题1：涨跌停判断使用周频数据（致命缺陷）

**现状**：用"本周收盘价 vs 上周收盘价涨幅>10%"来判断涨停。

**问题**：涨跌停是日频现象。周涨幅10%可能是每天涨2%的正常走势，根本不是涨停。反之，周一涨停但周五回落，代码会错误认为可以买入。

**修复方案**：在周频回测中，**放弃逐日涨跌停判断，改用简化规则**。

**方案A（推荐，最简单）**：直接移除涨跌停约束，在论文中声明：

> "由于本文使用周频截面数据调仓，无法精确判断每日涨跌停状态。考虑到周频调仓频率较低，单日涨跌停对周度持仓的影响有限，本文在回测中未施加涨跌停约束。这一简化可能导致回测收益略微高估，但不影响方法间的相对比较。"

**方案B（更严谨）**：接入日频数据，在调仓日检查当日是否涨停/跌停：

```python
def check_daily_limit(stock_code: str, date: str, daily_df: pd.DataFrame) -> dict:
    """
    检查某只股票在指定日期的涨跌停状态。

    参数:
        stock_code: 股票代码
        date: 调仓日期（YYYY-MM-DD）
        daily_df: 日频行情数据，需包含 open, close, high, low, prev_close

    返回:
        {"buy_blocked": bool, "sell_blocked": bool}
    """
    row = daily_df[(daily_df["stock_code"] == stock_code) & (daily_df["date"] == date)]
    if row.empty:
        return {"buy_blocked": True, "sell_blocked": True}

    prev_close = row["prev_close"].iloc[0]
    close = row["close"].iloc[0]

    limit_pct = _get_limit_pct(stock_code)
    limit_up = prev_close * (1 + limit_pct)
    limit_down = prev_close * (1 - limit_pct)

    buy_blocked = close >= limit_up - 1e-9
    sell_blocked = close <= limit_down + 1e-9
    return {"buy_blocked": buy_blocked, "sell_blocked": sell_blocked}

def _get_limit_pct(stock_code: str) -> float:
    """
    根据股票代码判断涨跌幅限制比例。

    参数:
        stock_code: 股票代码

    返回:
        涨跌幅限制比例（如0.10表示10%）
    """
    if stock_code.startswith("688"):
        return 0.20
    elif stock_code.startswith("300") or stock_code.startswith("301"):
        return 0.20
    elif stock_code.startswith("ST") or "ST" in stock_code:
        return 0.05
    else:
        return 0.10
```

**我的建议**：先用方案A快速推进，论文投稿前如果有时间再改方案B。方案A在学术界是可以接受的——很多顶刊论文也在周频/月频回测中省略了涨跌停约束。

## 3.2 问题2：停牌股票处理缺失（严重缺陷）

**现状**：持仓股票不在截面中时，直接从positions字典删除（市值归零）。

**问题**：停牌股票的持仓价值不会归零，而是保持停牌前最后价格。当前处理方式会导致净值在停牌周出现人为的断崖式下跌。

**修复方案**：

```python
# 原代码（backtest.py 第328-330行）：
# for code in list(positions):
#     if code not in px.index:
#         del positions[code]     # ← 直接删除，市值归零

# 修复后：
for code in list(positions):
    if code not in px.index:
        # 保留持仓，使用最后已知价格估值
        if code in last_known_price:
            positions[code]["price"] = last_known_price[code]
        # 不删除，保留在持仓中
    else:
        last_known_price[code] = px.loc[code, "close"]
```

**同时需要在市值计算中包含这些停牌股票**。

## 3.3 问题3：缺少基准对比（必须补充）

**现状**：回测结果只有策略净值，没有与市场基准的对比。

**问题**：没有基准对比，审稿人无法判断策略是否真的跑赢了市场。年化收益10%在大牛市中可能远跑输指数。

**修复方案**：在回测结果中增加基准净值曲线：

```python
def add_benchmark(backtest_result: pd.DataFrame, benchmark_name: str = "hs300") -> pd.DataFrame:
    """
    在回测结果中添加基准指数净值曲线。

    参数:
        backtest_result: 回测结果DataFrame，需包含date列
        benchmark_name: 基准名称，"hs300"或"zz500"

    返回:
        添加了基准净值列的DataFrame
    """
    import akshare as ak
    dates = backtest_result["date"]
    start_date = dates.min().strftime("%Y%m%d")
    end_date = dates.max().strftime("%Y%m%d")

    if benchmark_name == "hs300":
        df = ak.index_zh_a_hist(symbol="000300", period="weekly",
                                start_date=start_date, end_date=end_date)
    elif benchmark_name == "zz500":
        df = ak.index_zh_a_hist(symbol="000905", period="weekly",
                                start_date=start_date, end_date=end_date)

    df["date"] = pd.to_datetime(df["日期"])
    df["benchmark_nav"] = df["收盘"] / df["收盘"].iloc[0]
    result = backtest_result.merge(df[["date", "benchmark_nav"]], on="date", how="left")
    result["benchmark_nav"] = result["benchmark_nav"].ffill()
    return result
```

### 评估指标

| 指标 | 为什么需要 |
|------|-----------|
| NDCG@10 | 排序质量（预期略降，因为优化目标变了） |
| 年化收益 | 投资回报（预期提升） |
| 夏普比率 | 风险调整收益（预期显著提升） |
| 最大回撤 | 尾部风险（预期改善） |
| 换手率 | 交易成本敏感度 |

### 关键论证逻辑

**这是你论文最核心的实验，必须讲清楚一个逻辑**：

> 标准NDCG损失优化的是"排序质量"，但投资收益取决于Top-K股票的绝对收益水平。一个NDCG很高的模型可能选出的Top股票收益都很低（只是相对排序正确）。收益-夏普感知损失直接优化投资目标，因此NDCG可能略降（排序质量牺牲），但投资绩效显著提升（夏普比率、回撤改善）。

**这个"NDCG略降但夏普提升"的结果恰恰是你论文最有说服力的发现**——它证明排序质量和投资绩效不是一回事，优化排序质量≠优化投资收益。

### 替代方案：修改标签权重（更简单）

```python
def return_aware_relevance(y: np.ndarray, group_sizes: np.ndarray,
                           max_label: int = 30, alpha: float = 1.0) -> np.ndarray:
    """
    收益感知的relevance标签构造。

    与标准方法不同，relevance不仅取决于排名位置，
    还取决于实际收益率的大小。

    参数:
        y: 未来一周收益率
        group_sizes: 每个截面的样本数
        max_label: 最大标签值
        alpha: 收益敏感度参数，alpha越大，高收益股票的标签越高

    返回:
        离散relevance标签数组
    """
    rel = np.empty(len(y), dtype=np.int32)
    offset = 0
    for gsz in group_sizes:
        seg = y[offset:offset + gsz]
        order = np.argsort(-seg, kind="mergesort")
        rank = np.empty(gsz, dtype=np.int64)
        rank[order] = np.arange(gsz)

        # 标准排名标签
        rank_rel = (max_label * (1.0 - rank / max(gsz - 1, 1))).round().astype(np.int32)

        # 收益率加权：高收益股票的标签额外提升
        return_bonus = np.abs(seg) ** alpha
        return_bonus = max_label * 0.3 * return_bonus / (return_bonus.max() + 1e-9)

        combined = rank_rel + return_bonus
        combined = np.clip(combined, 0, max_label).round().astype(np.int32)
        rel[offset:offset + gsz] = combined
        offset += gsz
    return rel
```

## 4.3 实验2：多模型排序融合对比

### 实验组

| 组别 | 方法 | 说明 |
|------|------|------|
| B-LGBM | LightGBM单模型 | 基线 |
| B-XGB | XGBoost单模型 | 基线 |
| E2a | 分数平均融合 | 两模型预测分数归一化后取平均 |
| E2b | RRF融合 | Reciprocal Rank Fusion，k=60 |
| E2c | Stacking融合 | Ridge元学习器 |
| E2d | 加权RRF | 验证集NDCG加权 |

### RRF融合实现（核心代码）

```python
def reciprocal_rank_fusion(rankings: list, k: int = 60, weights: list = None) -> list:
    """
    Reciprocal Rank Fusion 排序融合。

    参数:
        rankings: 多个模型的排名列表，每个元素为股票代码的有序列表
        k: 平滑常数（默认60）
        weights: 各模型的权重（默认等权）

    返回:
        融合后的股票代码排序列表
    """
    if weights is None:
        weights = [1.0] * len(rankings)

    scores = {}
    for ranking, w in zip(rankings, weights):
        for rank_pos, stock_code in enumerate(ranking, start=1):
            if stock_code not in scores:
                scores[stock_code] = 0.0
            scores[stock_code] += w / (k + rank_pos)

    sorted_stocks = sorted(scores.items(), key=lambda x: x[1], reverse=True)
    return [code for code, _ in sorted_stocks]
```

### 关键论证逻辑

> RRF融合不依赖预测分数的绝对大小和分布，仅使用排名位置，因此对两个模型输出分数的尺度差异具有天然鲁棒性。此外，RRF的排名稳定性更高（每周排名变化更小），从而降低换手率和交易成本。

## 4.4 实验3：LLM可解释推荐评估

### 实验组

| 组别 | 推荐方式 | 说明 |
|------|---------|------|
| B-RAW | 纯模型输出 | Top-K股票代码+得分 |
| E3a | 模型+特征重要性 | 增加Top因子名称 |
| E3b | 模型+特征贡献分解 | 增加SHAP近似贡献 |
| E3c | 模型+特征贡献+LLM推荐 | 完整链路 |

### 评估方法

LLM推荐难以用传统指标量化，采用以下评估方式：

**定量评估**：
- 模拟执行收益：按LLM建议的买入/卖出操作执行，计算收益
- 与纯模型推荐的模拟收益对比

**定性评估**：
- 邀请3-5名有投资经验者对推荐合理性打分（1-5分）
- 评估维度：可理解性、可操作性、风险提示及时性

**一致性评估**：
- LLM建议操作与模型排序信号的一致率
- 如果LLM建议卖出模型排名靠前的股票，记录并分析原因

### 论文中的呈现方式

这个实验在论文中**不需要大量定量结果**，1-2个案例展示 + 定性讨论即可。因为LLM推荐的价值在于"可解释性"而非"收益提升"——底层信号相同，收益自然接近。

# 五、统计检验方法

## 5.1 为什么需要统计检验？

**审稿人最常问的问题**："你的方法比基线好，但这个差异是真实的还是随机的？"

仅报告数字大小是不够的。年化收益15% vs 12%，看起来有提升，但如果置信区间是[8%, 22%] vs [6%, 18%]，两者重叠严重，差异并不显著。

## 5.2 必须做的3个统计检验

### 检验1：Bootstrap置信区间

**目的**：为策略的核心指标构建置信区间，回答"这个夏普比率有多可靠？"

```python
import numpy as np

def bootstrap_metric(returns: np.ndarray, metric_fn, n_bootstrap: int = 5000,
                     confidence: float = 0.95, seed: int = 42) -> dict:
    """
    对任意策略指标构建Bootstrap置信区间。

    参数:
        returns: 周度收益率序列
        metric_fn: 指标计算函数，输入收益率序列，输出标量
        n_bootstrap: 重采样次数
        confidence: 置信水平
        seed: 随机种子

    返回:
        {"point": 点估计, "lower": 下界, "upper": 上界}
    """
    rng = np.random.default_rng(seed)
    point = metric_fn(returns)
    samples = []
    for _ in range(n_bootstrap):
        idx = rng.choice(len(returns), size=len(returns), replace=True)
        samples.append(metric_fn(returns[idx]))
    samples = np.array(samples)
    alpha = (1 - confidence) / 2
    return {
        "point": point,
        "lower": np.percentile(samples, alpha * 100),
        "upper": np.percentile(samples, (1 - alpha) * 100)
    }

# 使用示例
def sharpe_ratio(returns, annual_factor=52):
    return np.mean(returns) / np.std(returns, ddof=1) * np.sqrt(annual_factor)

result = bootstrap_metric(weekly_returns, sharpe_ratio)
print(f"夏普比率: {result['point']:.3f} [{result['lower']:.3f}, {result['upper']:.3f}]")
```

**在论文中的呈现方式**：

| 方法 | 夏普比率 | 95%置信区间 |
|------|---------|------------|
| B-LGBM | 1.23 | [0.85, 1.58] |
| E1b-LGBM | **1.56** | [1.12, 1.95] |
| E2b-RRF | **1.71** | [1.28, 2.10] |

如果置信区间不重叠，说明差异显著。

### 检验2：配对t检验 / Wilcoxon检验

**目的**：回答"方法A和方法B的周度收益差异是否统计显著？"

```python
from scipy import stats

def paired_significance_test(returns_a: np.ndarray, returns_b: np.ndarray,
                             alpha: float = 0.05) -> dict:
    """
    配对显著性检验：检验两种方法的周度收益差异是否显著。

    参数:
        returns_a: 方法A的周度收益率序列
        returns_b: 方法B的周度收益率序列
        alpha: 显著性水平

    返回:
        {"t_stat": t统计量, "p_value": p值, "significant": 是否显著,
         "method": "parametric"或"non-parametric"}
    """
    diff = returns_a - returns_b

    # 先检验差值是否服从正态分布
    _, p_normal = stats.shapiro(diff[:5000]) if len(diff) > 5000 else stats.shapiro(diff)

    if p_normal > 0.05:
        t_stat, p_value = stats.ttest_rel(returns_a, returns_b)
        method = "paired-t"
    else:
        t_stat, p_value = stats.wilcoxon(returns_a, returns_b)
        method = "wilcoxon"

    return {
        "t_stat": t_stat,
        "p_value": p_value,
        "significant": p_value < alpha,
        "method": method
    }
```

**在论文中的呈现方式**：

在结果表格中，用星号标注显著性：

| 方法 | 年化收益 | 夏普比率 | vs B-LGBM p值 |
|------|---------|---------|--------------|
| B-LGBM | 12.3% | 1.23 | — |
| E1b-LGBM | 15.8%** | 1.56** | 0.032 |
| E2b-RRF | 17.2%*** | 1.71*** | 0.008 |

> * p<0.1, ** p<0.05, *** p<0.01

### 检验3：Deflated Sharpe Ratio（推荐但非必须）

**目的**：考虑你做了多次超参搜索/多组实验的事实，校正夏普比率

```python
from scipy.stats import norm

def deflated_sharpe_ratio(sharpe: float, n_trials: int, skew: float,
                          kurtosis: float, n_obs: int) -> float:
    """
    计算Deflated Sharpe Ratio，校正多重试验导致的夏普比率虚高。

    参数:
        sharpe: 观测到的年化夏普比率
        n_trials: 独立试验次数（如超参搜索的trial数）
        skew: 收益率分布的偏度
        kurtosis: 收益率分布的超额峰度
        n_obs: 观测期数

    返回:
        DSR值（0-1之间，>0.95表示夏普比率可信）
    """
    # 夏普比率的标准误（考虑偏度和峰度）
    se = np.sqrt((1 - skew * sharpe + (kurtosis - 1) / 4 * sharpe**2) / (n_obs - 1))

    # 在n_trials次独立试验中最大夏普比率的期望
    expected_max = (1 - np.euler_gamma) * norm.ppf(1 - 1/n_trials) + np.euler_gamma * norm.ppf(1 - 1/(n_trials * np.e))

    # DSR = P(真实夏普 > 0 | 观测数据)
    dsr = norm.cdf((sharpe - expected_max) / se)
    return dsr
```

**在论文中的呈现方式**：

> "考虑到超参搜索进行了20次独立试验，本文使用Deflated Sharpe Ratio（Bailey & Lopez de Prado, 2014）校正多重试验的影响。校正后，所提方法的DSR为0.97，表明夏普比率在统计上是可信的。"

## 6.3 多空组合分析

**目的**：证明策略赚取的是Alpha（选股能力），而非Beta（市场暴露）。

```python
def quantile_portfolio_returns(scores: pd.DataFrame, n_quantiles: int = 5) -> dict:
    """
    按模型得分将股票分为N个分位数组，计算每组的等权组合收益。

    参数:
        scores: 包含 date, stock_code, score, future_return_1w 的DataFrame
        n_quantiles: 分组数量

    返回:
        {"Q1": 收益率序列, "Q2": ..., "Q5": ..., "LS": Q1-Q5}
    """
    result = {}
    for q in range(1, n_quantiles + 1):
        label = f"Q{q}"
        result[label] = []

    for date, group in scores.groupby("date"):
        group = group.sort_values("score", ascending=False)
        quantile_size = len(group) // n_quantiles
        for q in range(1, n_quantiles + 1):
            if q == 1:
                stocks = group.iloc[:quantile_size]
            elif q == n_quantiles:
                stocks = group.iloc[(q-1)*quantile_size:]
            else:
                stocks = group.iloc[(q-1)*quantile_size:q*quantile_size]
            result[f"Q{q}"].append(stocks["future_return_1w"].mean())

    for key in result:
        result[key] = np.array(result[key])
    result["LS"] = result["Q1"] - result[f"Q{n_quantiles}"]
    return result
```

**在论文中的呈现方式**：

| 分组 | 年化收益 | 夏普比率 |
|------|---------|---------|
| Q1（最高分） | 18.5% | 1.65 |
| Q2 | 12.3% | 1.12 |
| Q3 | 8.7% | 0.78 |
| Q4 | 5.2% | 0.45 |
| Q5（最低分） | 2.1% | 0.18 |
| **多空（Q1-Q5）** | **16.4%** | **1.42** |

**关键**：如果Q1到Q5的收益单调递减，说明模型排序确实有效。如果Q1和Q5差不多，说明模型没有真正的选股能力。

# 七、消融实验设计

## 7.1 什么是消融实验？

消融实验（Ablation Study）是**逐步移除你的创新组件**，观察效果下降多少，从而证明每个组件都有贡献。

**这是审稿人判断你创新点是否真实有效的核心依据。**

## 7.2 消融实验设计

| 实验编号 | 损失函数 | 模型融合 | LLM推荐 | 说明 |
|---------|---------|---------|---------|------|
| Full | ✅ 收益-夏普感知 | ✅ RRF | ✅ 有 | 完整方法 |
| w/o Loss | ❌ 标准NDCG | ✅ RRF | ✅ 有 | 移除创新点① |
| w/o Fusion | ✅ 收益-夏普感知 | ❌ 单模型 | ✅ 有 | 移除创新点②的融合部分 |
| w/o LLM | ✅ 收益-夏普感知 | ✅ RRF | ❌ 无 | 移除创新点②的LLM部分 |
| w/o All | ❌ 标准NDCG | ❌ 单模型 | ❌ 无 | 退化为基线 |

**预期结果**：

| 实验 | 夏普比率 | vs Full变化 |
|------|---------|------------|
| Full | 1.71 | — |
| w/o Loss | 1.45 | -0.26（损失函数贡献） |
| w/o Fusion | 1.52 | -0.19（融合贡献） |
| w/o LLM | 1.68 | -0.03（LLM贡献较小，主要提升可解释性） |
| w/o All | 1.23 | -0.48（全部创新的累计贡献） |

**关键**：每个组件移除后效果都应下降，证明每个创新点都有独立贡献。
