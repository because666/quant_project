# 论文协作方案

> 你：算法、实验、代码、数据
> 合作者：文献整理、论文写作、排版、格式修改

---

# 一、协作总览

## 1.1 需要共享什么

你们不需要"一起写代码"，但需要在这些方面协作：

| 协作内容 | 谁产出 | 谁接收 | 交换媒介 |
|---------|--------|--------|---------|
| 实验进度 | 你 | 合作者 | 看板 + 每周同步 |
| 实验结果（数据+图表） | 你 | 合作者 | 腾讯文档 / 石墨文档 |
| 算法原理描述 | 你 | 合作者 | 腾讯文档 |
| 文献笔记 | 合作者 | 你 | 腾讯文档 |
| 论文各章节草稿 | 合作者 | 你 | Overleaf（推荐） |
| 参考论文PDF | 合作者 | 双方 | 共享网盘 / 微信群 |

## 1.2 协作工具推荐

| 用途 | 推荐工具 | 替代方案 | 说明 |
|------|---------|---------|------|
| **论文写作** | Overleaf (overleaf.com) | 腾讯文档 | Overleaf是在线LaTeX编辑器，支持实时协作，自动版本管理。你和合作者可以同时编辑 |
| **文献管理** | Zotero + 群组共享 | Mendeley | 合作者整理文献，共享到群组，双方都能看到 |
| **进度看板** | Trello / 飞书多维表格 | 腾讯文档表格 | 跟踪每周任务完成状态 |
| **实验结果共享** | 腾讯文档 / 石墨文档 | 石墨文档 | 你把实验结果填入共享表格，合作者直接引用 |
| **日常沟通** | 微信/QQ | — | 每天或隔天小同步 |
| **代码仓库** | GitHub/Gitee ✅ (已有) | — | 你管理代码，合作者不需要动 |

# 二、推荐使用 Overleaf 写论文

## 2.1 为什么选择 Overleaf

| 优势 | 说明 |
|------|------|
| 无需安装LaTeX | 网页版直接编辑，浏览器打开即用 |
| 实时协作 | 你和合作者可以同时编辑，看到对方光标 |
| 版本管理 | 自动保存历史版本，误删可恢复 |
| 模板丰富 | 内置各大会议模板（ICAIF/CIKM/IJCAI等） |
| 评论功能 | 可以给段落加批注，@对方 |
| 图表上传 | 你制作的图表直接上传，合作者在论文中引用 |

## 2.2 在Overleaf上的分工

Overleaf只有一个项目，但你们**不要同时编辑同一章节**，按以下方式分工：

| 章节 | 谁写初稿 | 谁修改 | 说明 |
|------|---------|--------|------|
| 1. 引言 (Introduction) | **合作者** | 你 | 合作者写背景和动机，你补充技术难点 |
| 2. 相关工作 (Related Work) | **合作者** | 你 | 合作者写文献综述，你补充技术细节 |
| 3. 方法 (Methodology) | **你** | 合作者 | 你写核心算法，合作者润色语言 |
| 4. 实验 (Experiments) | **你** | 合作者 | 你填数据和结果，合作者润色 |
| 5. 结论 (Conclusion) | **合作者** | 你 | 合作者写总结，你确认技术贡献 |
| 摘要 (Abstract) | 双方 | 双方 | 最后一起写 |

## 2.3 Overleaf项目设置建议

1. 注册 Overleaf 账号（免费即可）
2. 创建新项目，选择目标会议模板（如 ICAIF）
3. 邀请合作者加入（右上角 Share → 输入邮箱）
4. 项目设置中开启：
   - Track Changes（追踪修改）
   - 开启 Overleaf v2 编辑器
5. 合作者负责：上传文献BibTeX文件、设置格式、排版

# 三、协作流程：从开始到投稿

## 3.1 分阶段协作模式

根据论文写作的不同阶段，协作的侧重点不同：

### 第一阶段（第1-2周）：各自准备

| 你做的事 | 合作者做的事 |
|---------|------------|
| 修复环境、跑通基线 | 阅读第一阶段必读论文（B1-B5） |
| 实现损失函数改进 | 整理文献笔记，共享到群组 |
| | 创建 Overleaf 项目，设置模板 |
| | 整理 BibTeX 文献引用文件 |

**交换的东西**：
- 你：无需输出，专注代码
- 合作者：文献笔记链接、Overleaf项目链接

### 第二阶段（第3-4周）：实验为主

| 你做的事 | 合作者做的事 |
|---------|------------|
| 跑全部实验（Exp1-4） | 阅读第二阶段论文（C1-C11） |
| 记录所有实验结果 | 在 Overleaf 中写 Related Work 初稿 |
| 发现问题及时沟通 | 补充满意文献 |

**交换的东西**：
- 你：每周五把实验结果填入共享表格
- 合作者：Related Work 初稿链接、需要你补充说明的标注

### 第三阶段（第5-6周）：结果整理 + 论文初稿

| 你做的事 | 合作者做的事 |
|---------|------------|
| 制作所有图表（Fig.1-9, Tab.1-6） | 在 Overleaf 写 Introduction 和 Conclusion |
| 在 Overleaf 写 Method 部分 | 整理论文格式、引用 |
| 在 Overleaf 填 Experiment 数据 | 语言润色 |

**交换的东西**：
- 你：图表PDF文件上传到 Overleaf，Method部分文字
- 合作者：Introduction + Conclusion 草稿，格式检查

### 第四阶段（第7-10周）：修改 + 投稿

| 你做的事 | 合作者做的事 |
|---------|------------|
| 检查技术正确性 | 全文排版 |
| 确认实验数据无误 | 调整格式到会议模板 |
| 审稿意见回应 | 回复审稿意见（如果有） |

## 3.2 每周同步节奏

```
周一：合作者发本周写作计划 + 需要你提供的资料清单
      ↓
周三前：你提供合作者需要的资料（实验结果、图表、技术描述）
      ↓
周五：双方腾讯会议/语音通话，同步进展（15-30分钟）
      你的汇报：实验进度、发现的问题、下一周计划
      合作者汇报：写作进度、需要你补充的内容

如果卡住了（比如实验跑不出来），随时在群里说，不要等到周五。
```

## 3.3 第一次同步会议（你俩应该尽快开的）

第一次会议要敲定这些事情：

1. **目标会议**：ICAIF（6-8月截稿）还是 CIKM（5月截稿）？查今年的截稿日期
2. **Overleaf项目**：合作者创建，邀请你加入
3. **共享表格**：创建腾讯文档/石墨文档，用来传实验结果
4. **Zotero群组**：合作者创建，双方加入
5. **本周计划**：
   - 你：修复环境 + 跑通基线
   - 合作者：读B1-B5论文 + 整理文献笔记

## 4.3 方法部分的技术描述模板

> 你在写 Method 部分时，按这个模板把技术细节写给合作者

### 3.2 收益-夏普感知排序损失函数

**数学公式**：
```
L = -Σ_q (1/Z_q) Σ_k g(r_k) / log_2(k+1) + λ·max(0, -Sharpe_TopK)
其中 g(r_k) = |r_k|^α · sign(r_k) + C
```

**关键参数**：α=1.0, λ=0.1, C=1.0

**与标准NDCG的区别**：
- 标准NDCG：优化排序质量，不关心Top组合的绝对收益
- 本文方法：直接优化Top组合的投资绩效

**直觉解释**（帮合作者理解，她要用白话写进论文）：
> "标准NDCG只看排名对不对，但排名对了不代表能赚钱。比如排名前10的股票收益率可能都很低。我们的方法让模型不仅关心排名位置，还关心排名靠前的股票到底赚不赚钱。"

# 五、论文各部分的写作协作方式

## 5.1 具体谁写什么

**合作者独立写**：
- Related Work（她读文献她最熟）
- Conclusion（总结性内容）
- Abstract（最后写）

**你独立写**：
- Methodology（核心算法只有你懂）
- Experiments 中的数据和结果（你跑的数据你填）

**一起写**：
- Introduction：合作者写背景和动机，你写贡献点和技术难点
- Experiments 中的分析和讨论：合作者写文字，你确认技术表述的准确性

## 5.2 写作流程（按章节顺序）

```
第6周初：合作者写 Related Work 初稿 → 你审阅 → 修改
第6周末：你写 Method 初稿 → 合作者润色
第7周初：合作者写 Introduction 初稿 → 你补充技术背景
第7周中：你填 Experiment 数据 → 合作者写分析文字
第7周末：合作者写 Conclusion → 你确认
第8周：合作者整理 Abstract + 全文格式 + 引用
第8周末：双方通读全文，互相批注
第9周：修改，定稿
第10周：投稿
```

## 5.3 Overleaf上的协作规范

1. **每章节由一个负责人写初稿**，另一个修改
2. **不要同时编辑同一段落**，避免冲突
3. 用 Overleaf 的 **评论功能**（选中文字 → 加评论）而不是在群里发截图
4. 用 **标注 TODO** 标记未完成的部分：
   ```
   % TODO: 需要等实验跑完补充数据
   ```
5. 每周五合并双方的修改，处理冲突

# 六、Git仓库协作方式

## 6.1 你管代码，她不管

| 内容 | 放哪儿 | 谁更新 |
|------|--------|--------|
| Python代码 | Git仓库 | **你** |
| 实验数据 | Git仓库（.gitignore除外） | **你** |
| 实验结果CSV/JSON | Git仓库 | **你** |
| 图表PDF/PNG | Git仓库 + Overleaf | **你**上传 |
| 论文LaTeX | Overleaf | **合作者** |
| 文献BibTeX | Overleaf | **合作者** |
| 文献笔记 | 共享表格 | **合作者** |

## 6.2 图的传递方式

你的图表放在 Git 仓库里，但合作者写论文时需要在 Overleaf 中用这些图：

```
你：制作图表 → 保存为PDF → 提交到Git仓库
                         ↓
你：从Git下载 → 上传到 Overleaf 项目
                         ↓
合作者：在 Overleaf 中 \includegraphics{fig1.pdf}
```

**或者更简单**：你用 Overleaf 的"文件上传"功能，直接上传图表PDF到 Overleaf 项目里。合作者直接在 Overleaf 里用。

# 七、第一次同步会议的议程

> 建议你们尽快开一次视频会议（30分钟），敲定以下事项：

## 会议议程

1. **自我介绍**（如果还不熟的话）
2. **论文方向确认**（5分钟）
   - 论文标题、创新点、目标会议
   - 我发之前的文档你看了吗
3. **工具搭建**（10分钟）
   - 创建 Overleaf 项目（合作者当场操作）
   - 创建共享表格（你来操作）
   - 加微信/QQ群
4. **本周计划**（10分钟）
   - 你的计划：修复环境 + 跑通基线
   - 她的计划：读B1-B5论文 + 创建Overleaf项目 + 开始整理文献笔记
   - 约定第一次同步时间（比如下周一）
5. **下一步**（5分钟）
   - 会后她发Overleaf链接给你
   - 你发共享表格链接给她
   - 建立Zotero群组

## 会议前你需要准备

- [ ] Overleaf注册账号（如还没有）
- [ ] 创建一个腾讯文档/石墨文档的共享表格
- [ ] 确认目标会议的截稿日期
- [ ] 看一下共享表格模板，提前填好实验进度那一行

## 会议后你俩各自要做的

**你**：
- [ ] 把共享表格链接发给合作者
- [ ] 按行动计划开始第一周工作
- [ ] 每周五填实验结果到共享表格

**合作者**：
- [ ] 把 Overleaf 项目链接发给你
- [ ] 开始读 B1-B5 论文
- [ ] 在共享表格中建立文献阅读记录表
- [ ] 整理 BibTeX 文件上传到 Overleaf

# 附录：合作者可能需要的帮助

合作者不懂代码和量化，她写论文时可能需要你解释以下概念：

| 她可能会问 | 你的回答要点 |
|-----------|------------|
| "什么是排序学习？" | 不是预测具体收益率，而是给股票排序。类似搜索排序 |
| "什么是NDCG？" | 衡量排序质量的指标，比如前十名有几个选对了 |
| "什么是RRF融合？" | 两个模型的排名取平均，类似考试的两个阅卷老师打分取平均 |
| "为什么夏普比率重要？" | 衡量"每承担一分风险能赚多少钱"，比只看收益更全面 |
| "损失函数是啥？" | 模型训练的"目标"，告诉模型什么方向是好的 |

**建议**：合作者写论文时如果对某些技术表述不确定，在 Overleaf 中加评论标注，你来确认修改。
